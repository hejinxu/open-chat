# 多存储后端实现计划

> 本文档记录将消息存储从 localStorage 扩展到支持 SQLite 和 PostgreSQL 的实现计划。

---

## 一、需求清单

| # | 需求 | 设计决策 |
|---|------|----------|
| 1 | 同一会话按时序保存不同智能体对话 | 消息表结构不变，`agent_id` + `created_at` |
| 2 | 同一会话可随时切换智能体 | 保持现有切换 effect 逻辑 |
| 3 | 同一会话同一智能体保持后端会话ID | `ConversationRecord.agents[agentId].backend_conversation_id` |
| 4 | 切换智能体重新获取参数，未完成不允许发送 | `promptConfig` 守卫 + 10秒超时降级到 localStorage |
| 5 | 页面加载获取默认智能体参数 | init 时获取，10秒超时降级 |
| 6 | 无参数智能体（直连LLM） | `isDirectLLM` 跳过参数获取，`promptConfig = { prompt_variables: [] }` |
| 7 | 直连LLM保持上下文 | 从 `chatList` 构建 OpenAI 格式 messages 数组（包含所有智能体对话） |

---

## 二、设计决策

| 问题 | 决策 |
|------|------|
| **总原则** | **以远程存储为主，本地存储只是远程存储失效时的备份来源** |
| 存储后端切换策略 | 环境变量 `NEXT_PUBLIC_STORAGE_BACKEND` 配置，部署时确定 |
| SQLite 数据库路径 | 项目根目录 `data/openchat.db` |
| 数据迁移 | 不迁移，新存储从空白开始 |
| 多标签页并发 | 添加全局锁机制，同一时间只允许一个标签页写入 |
| 锁等待策略 | 等待并重试 |
| 删除操作 | 先删远程，成功后再删本地 |
| 消息列表 | 一次性加载所有消息 |
| 数据清理 | 不清理，永久保留 |
| 会话列表排序 | 按 `updated_at` 降序 |
| 保存失败处理 | 优先写远程，成功后写本地；失败降级写本地 + toast 提示 |
| 初始化流程 | 优先远程获取，失败降级到本地 |
| `onData` 中保存会话ID | 只对 Dify 类型智能体，第一个 chunk 触发；写远程成功后写本地 |
| 错误提示 | 显示 toast 提示 |

## 四、数据流设计

### 写操作流程

```
saveAgentParams / saveBackendConvId / saveMessage
     │
     ├─→ 1. 立即更新 ref 缓存（内存，同步）
     │
     ├─→ 2. 获取全局写锁
     │
     └─→ 3. 优先写入远程存储（主存储）
             │
             ├─→ 成功：
             │     ├─→ 释放锁
             │     └─→ 写入 localStorage 缓存
             │
             └─→ 失败：
                   ├─→ 释放锁
                   ├─→ 降级写入 localStorage
                   ├─→ 标记待重试
                   └─→ 显示 toast 提示
```

### 读操作流程

```
getAgentParams / getBackendConvId / getMessages
     │
     ├─→ 1. ref 缓存（同步，最快）
     │
     ├─→ 2. 远程存储（异步，主存储，10s 超时）
     │       │
     │       ├─→ 成功：
     │       │     ├─→ 更新 localStorage 缓存
     │       │     └─→ 返回数据
     │       │
     │       └─→ 超时/失败：
     │             ├─→ 降级到 localStorage
     │             └─→ 显示 toast 提示
     │
     └─→ 3. localStorage（同步，备份）
```

### 删除操作流程

```
deleteConversation / deleteMessages
     │
     ├─→ 1. 获取全局写锁
     │
     ├─→ 2. 优先删除远程存储（主存储）
     │       │
     │       ├─→ 成功：
     │       │     ├─→ 删除 localStorage 缓存
     │       │     └─→ 释放锁
     │       │
     │       └─→ 失败：
     │             ├─→ 释放锁
     │             ├─→ 不删 localStorage
     │             ├─→ 标记待重试
     │             └─→ 显示 toast 提示
     │
     └─→ 3. 更新 ref 缓存
```

### 初始化流程

```
页面加载
     │
     ├─→ 1. 从远程存储获取会话列表（主存储）
     │       │
     │       ├─→ 成功：
     │       │     ├─→ 更新 localStorage 缓存
     │       │     └─→ 显示远程数据
     │       │
     │       └─→ 失败/超时：
     │             ├─→ 降级到 localStorage
     │             └─→ 显示本地数据 + toast 提示
     │
     └─→ 2. 更新 ref 缓存
```

### 6.3 全局写锁

```typescript
// lib/storage/tab-lock.ts
const LOCK_KEY = 'open_chat_tab_lock'
const LOCK_TIMEOUT = 5000  // 5 秒超时
const MAX_WAIT_TIME = 10000  // 最大等待 10 秒

export class TabLock {
  private tabId: string
  private lockInterval: NodeJS.Timeout | null = null

  constructor() {
    this.tabId = `tab_${Date.now()}_${Math.random().toString(36).slice(2, 8)}`
  }

  async acquireLock(): Promise<boolean> {
    const startTime = Date.now()
    
    while (true) {
      const lockData = this.getLock()
      
      if (!lockData || lockData.tabId === this.tabId || Date.now() - lockData.timestamp > LOCK_TIMEOUT) {
        this.setLock()
        this.lockInterval = setInterval(() => this.setLock(), LOCK_TIMEOUT / 2)
        return true
      }
      
      if (Date.now() - startTime > MAX_WAIT_TIME) {
        return false
      }
      
      await new Promise(resolve => setTimeout(resolve, 100))
    }
  }

  releaseLock(): void {
    if (this.lockInterval) {
      clearInterval(this.lockInterval)
      this.lockInterval = null
    }
    const lockData = this.getLock()
    if (lockData && lockData.tabId === this.tabId) {
      localStorage.removeItem(LOCK_KEY)
    }
  }

  private getLock(): { tabId: string; timestamp: number } | null {
    try {
      const data = localStorage.getItem(LOCK_KEY)
      return data ? JSON.parse(data) : null
    } catch {
      return null
    }
  }

  private setLock(): void {
    localStorage.setItem(LOCK_KEY, JSON.stringify({
      tabId: this.tabId,
      timestamp: Date.now()
    }))
  }
}

export const tabLock = new TabLock()
```

### 6.4 RemoteStorageProvider

```typescript
// lib/storage/remote-storage.ts
import { tabLock } from './tab-lock'
import Toast from '@/app/components/base/toast'

export class RemoteStorageProvider implements StorageProvider {
  private localStorageProvider = new LocalStorageProvider()
  private baseUrl = '/api/storage'

  async getConversations(): Promise<ConversationRecord[]> {
    try {
      const res = await fetch(`${this.baseUrl}/conversations`)
      if (!res.ok) throw new Error('API failed')
      const data = await res.json()
      if (!data.success) throw new Error(data.error)
      
      await this.localStorageProvider.saveAllConversations(data.data)
      return data.data
    } catch (error) {
      console.warn('Remote storage failed, falling back to localStorage:', error)
      Toast.notify({ type: 'warning', message: '远程存储不可用，使用本地数据' })
      return this.localStorageProvider.getConversations()
    }
  }

  async saveConversation(conv: ConversationRecord): Promise<void> {
    const hasLock = await tabLock.acquireLock()
    if (!hasLock) {
      throw new Error('获取写锁超时')
    }

    try {
      const res = await fetch(`${this.baseUrl}/conversations`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(conv),
      })
      if (!res.ok) throw new Error('API failed')
      
      await this.localStorageProvider.saveConversation(conv)
    } catch (error) {
      console.error('Failed to save to remote:', error)
      Toast.notify({ type: 'error', message: '保存到远程失败，已降级到本地存储' })
      await this.localStorageProvider.saveConversation(conv)
      throw error
    } finally {
      tabLock.releaseLock()
    }
  }

  async deleteConversation(id: string): Promise<void> {
    const hasLock = await tabLock.acquireLock()
    if (!hasLock) {
      throw new Error('获取写锁超时')
    }

    try {
      const res = await fetch(`${this.baseUrl}/conversations`, {
        method: 'DELETE',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ id }),
      })
      if (!res.ok) throw new Error('API failed')
      
      await this.localStorageProvider.deleteConversation(id)
    } catch (error) {
      console.error('Failed to delete from remote:', error)
      Toast.notify({ type: 'error', message: '删除远程数据失败，本地数据已保留' })
      throw error
    } finally {
      tabLock.releaseLock()
    }
  }

  // ... 其他方法类似
}
```

### 6.5 异步获取 backend_conversation_id

```typescript
const backendConvIdCacheRef = useRef<Record<string, string>>({})

async function getBackendConvId(convId: string, agentId: string): Promise<string | null> {
  const cacheKey = `${convId}:${agentId}`
  
  if (backendConvIdCacheRef.current[cacheKey]) {
    return backendConvIdCacheRef.current[cacheKey]
  }
  
  if (storageBackend !== 'local') {
    const provider = getStorageProvider()
    const conv = await withTimeout(
      provider.getConversationById(convId),
      10000,
      null
    )
    if (conv) {
      const backendConvId = conv.agents?.[agentId]?.backend_conversation_id || null
      if (backendConvId) {
        backendConvIdCacheRef.current[cacheKey] = backendConvId
      }
      return backendConvId
    }
    Toast.notify({ type: 'warning', message: '获取会话ID超时，使用本地缓存' })
  }
  
  return getBackendConvIdFromLocalStorage(convId, agentId)
}
```

## 七、智能体类型与会话 ID

| 智能体类型 | `backend_type` | chunk 中返回 `conversation_id` | 需要保存 | 上下文保持方式 |
|------------|----------------|-------------------------------|----------|----------------|
| Dify | `dify` | ✅ 是（第一个 chunk） | ✅ 是 | 后端 `conversation_id` |
| 直连 LLM | `direct_llm` | ❌ 否（空值） | ❌ 否 | 前端 `messages` 数组（包含所有智能体对话） |
| FastGPT | `fastgpt` | 待确认 | 待确认 | 待确认 |
| n8n | `n8n` | 待确认 | 待确认 | 待确认 |

## 九、风险和缓解措施

| 风险 | 严重性 | 缓解措施 |
|------|--------|----------|
| 远程存储延迟导致用户体验下降 | 高 | 10s 超时 + localStorage 降级 + toast 提示 |
| 全局锁导致写操作阻塞 | 中 | 5s 锁超时 + 10s 最大等待 + toast 提示 |
| `onData` 中保存 backend_conv_id 频繁调用 | 中 | `hasSavedBackendConvId` 标志位 + Dify 类型判断 |
| localStorage 和远程数据不一致 | 低 | 以远程为准，本地只是缓存 |
| SQLite 并发写入限制 | 低 | WAL 模式 + 事务 |
| 网络不稳定导致写入失败 | 中 | 降级到 localStorage + toast 提示 |
