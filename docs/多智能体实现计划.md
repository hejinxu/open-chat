> **版本**: v1.0

> **作者**: opencode + user

---

## 一、目标

通过配置文件实现多智能体对接，支持前端选择智能体切换对话，后端支持 Dify、直连大模型、FastGPT、n8n 等多种后端类型。

## 二、核心设计决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 对话是否绑定智能体 | **不绑定** | 一个对话可包含多个智能体的消息 |
| 消息是否绑定智能体 | **绑定** | 每条消息记录发送时使用的智能体 |
| agentId 存储位置 | **组件状态** | 不放 URL，切换智能体不影响对话 |
| 智能体是否必选 | **可选** | 不选择则使用默认智能体 |
| 存储方式 | **localStorage（可替换）** | 通过 StorageProvider 接口抽象 |
| 后端适配 | **适配器模式** | 支持多种后端类型 |
| 类型命名 | **ConversationRecord / MessageRecord** | 不绑定存储方式，数据库友好 |

## 三、数据模型

### 3.1 ConversationRecord（对话记录）

```typescript
interface ConversationRecord {
  id: string                    // 本地生成的唯一 ID
  name: string                  // 对话标题
  created_at: number            // Unix 时间戳（秒）
  updated_at: number            // Unix 时间戳（秒）
}
```

### 3.2 MessageRecord（消息记录）

```typescript
interface MessageRecord {
  id: string                    // 本地生成的唯一 ID
  conversation_id: string       // 关联的对话 ID
  agent_id: string | null       // 发送时选择的智能体 ID（null = 默认智能体）
  agent_name: string | null     // 智能体名称（冗余，便于显示）
  role: 'user' | 'assistant'   // 消息角色
  content: string               // 消息内容
  is_answer: boolean            // 是否为 AI 回复
  feedback: { rating: string } | null  // 用户反馈
  message_files: any[]          // 附件
  agent_thoughts: any[]         // 思考过程
  created_at: number            // Unix 时间戳（秒）
}
```

### 3.3 AgentConfig（智能体配置）

```typescript
type BackendType = 'dify' | 'direct_llm' | 'fastgpt' | 'n8n'

interface AgentConfig {
  id: string
  name: string
  icon: string
  description: string
  backend_type: BackendType
  api_key: string
  api_url: string
  model?: string               // direct_llm 模式必填
  is_default: boolean
  is_enabled: boolean
  extra_config?: Record<string, any>
}

type AgentInfo = Omit<AgentConfig, 'api_key' | 'api_url'>
```

## 四、存储抽象层

### 4.1 StorageProvider 接口

```typescript
interface StorageProvider {
  getConversations(): Promise<ConversationRecord[]>
  getConversationById(id: string): Promise<ConversationRecord | null>
  saveConversation(conv: ConversationRecord): Promise<void>
  deleteConversation(id: string): Promise<void>

  getMessages(conversationId: string): Promise<MessageRecord[]>
  saveMessage(msg: MessageRecord): Promise<void>
  deleteMessages(conversationId: string): Promise<void>
}
```

### 4.2 LocalStorageProvider 实现

使用 localStorage 实现所有接口方法，数据序列化为 JSON。

### 4.3 未来 DatabaseProvider（预留）

使用 Prisma/PostgreSQL 实现，接口完全相同，只需替换实现。

## 五、服务层

### 5.1 ConversationService

- `getAllConversations()` - 获取所有对话
- `createConversation(name)` - 创建新对话
- `updateConversation(id, updates)` - 更新对话
- `deleteConversation(id)` - 删除对话及其消息

### 5.2 MessageService

- `getMessages(conversationId)` - 获取对话的消息列表（按时间排序）
- `saveMessage(msg)` - 保存消息
- `saveUserMessage(...)` - 保存用户消息
- `saveAssistantMessage(...)` - 保存 AI 回复

## 六、适配器层

### 6.1 ChatAdapter 接口

```typescript
interface ChatAdapter {
  type: string
  sendMessage(params: SendMessageParams): Promise<Response>
  stopMessage(taskId: string, user: string): Promise<void>
  getConversations(user: string): Promise<any[]>
  getMessages(conversationId: string, user: string): Promise<any[]>
  getParameters(user: string): Promise<any>
  renameConversation(id: string, name: string, user: string, autoGenerate?: boolean): Promise<any>
}
```

### 6.2 适配器实现

| 适配器 | 文件 | 后端 |
|--------|------|------|
| DifyAdapter | `lib/adapters/dify.ts` | Dify AI 平台 |
| LLMAdapter | `lib/adapters/llm.ts` | OpenAI 兼容 API |
| FastGPTAdapter | `lib/adapters/fastgpt.ts` | FastGPT（预留） |
| N8NAdapter | `lib/adapters/n8n.ts` | n8n Webhook（预留） |

### 6.3 适配器工厂

```typescript
function createAdapter(agent: AgentConfig): ChatAdapter {
  switch (agent.backend_type) {
    case 'dify': return new DifyAdapter(agent.api_key, agent.api_url)
    case 'direct_llm': return new LLMAdapter(agent.api_key, agent.api_url, agent.model)
    // ...
  }
}
```

### 8.3 发送消息 API 示例

```typescript
// POST /api/chat-messages
// Body: { query, conversation_id, agent_id, ... }

export async function POST(request: NextRequest) {
  const body = await request.json()
  const { query, conversation_id, agent_id, ...rest } = body

  const agent = agent_id ? getAgentById(agent_id) : getDefaultAgent()
  const adapter = createAdapter(agent)
  const { user } = getInfo(request)

  const response = await adapter.sendMessage({
    query, conversation_id, user,
    inputs: rest.inputs, files: rest.files,
  })

  return new Response(response.body, {
    headers: { 'Content-Type': 'text/event-stream' },
  })
}
```

## 十、数据流

```
用户发送消息（可选选择智能体）
  ↓
前端携带 agent_id 调用 /api/chat-messages
  ↓
后端根据 agent_id 获取 AgentConfig
  ↓
根据 backend_type 创建对应适配器
  ↓
适配器调用对应的后端 API
  ↓
返回 SSE 流
  ↓
前端统一处理响应（兼容不同格式）
  ↓
保存消息到本地存储（携带 agent_id）
  ↓
更新界面显示
```
