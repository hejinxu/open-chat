# 多智能体对话客户端实现计划

> **版本**: v1.0
> **状态**: 已实施
> **日期**: 2026-05-08（更新于 2026-05-09）
> **作者**: opencode + user

---

## 一、目标

通过配置文件实现多智能体对接，支持前端选择智能体切换对话，后端支持 Dify、直连大模型、FastGPT、n8n 等多种后端类型。

## 二、核心设计决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 对话是否绑定智能体 | **不绑定** | 一个对话可包含多个智能体的消息 |
| 消息是否绑定智能体 | **绑定** | 每条消息记录发送时使用的智能体 |
| agentId 存储位置 | **组件状态** | 不放 URL，切换智能体不影响对话 |
| 智能体是否必选 | **可选** | 不选择则使用默认智能体 |
| 存储方式 | **localStorage（可替换）** | 通过 StorageProvider 接口抽象 |
| 后端适配 | **适配器模式** | 支持多种后端类型 |
| 类型命名 | **ConversationRecord / MessageRecord** | 不绑定存储方式，数据库友好 |

## 三、数据模型

### 3.1 ConversationRecord（对话记录）

```typescript
interface AgentSessionState {
  params: Record<string, any>          // 该 Agent 在本会话中最后一次的参数值
  backend_conversation_id?: string     // 后端返回的会话 ID（如 Dify conversation_id）
}

interface ConversationRecord {
  id: string                    // 本地生成的唯一 ID
  name: string                  // 对话标题（取第一条消息前 30 字）
  created_at: number            // Unix 时间戳（秒）
  updated_at: number            // Unix 时间戳（秒）
  agents: Record<string, AgentSessionState>  // 每个智能体在本会话中的状态
}
```

**设计要点：**
- 一个本地会话的所有数据内聚在 `ConversationRecord` 中，不分散在多个 localStorage key
- `agents[agentId].params` 保存该智能体最后一次设置的参数值，用于下次发送时使用
- `agents[agentId].backend_conversation_id` 保存后端（如 Dify）返回的会话 ID，首次发送为 null，收到响应后保存
- 消息仍独立存储在 `MessageRecord[]` 中（`open_chat_messages` key），通过 `conversation_id` 关联

### 3.2 MessageRecord（消息记录）

```typescript
interface MessageRecord {
  id: string                    // 本地生成的唯一 ID
  conversation_id: string       // 关联的对话 ID
  agent_id: string | null       // 发送时选择的智能体 ID
  agent_name: string | null     // 智能体名称（冗余，便于显示）
  role: 'user' | 'assistant'   // 消息角色
  content: string               // 消息内容
  is_answer: boolean            // 是否为 AI 回复
  feedback: { rating: string } | null  // 用户反馈
  message_files: any[]          // 附件
  agent_thoughts: any[]         // 思考过程
  created_at: number            // Unix 时间戳（秒）
}
```

### 3.3 AgentConfig（智能体配置）

```typescript
type BackendType = 'dify' | 'direct_llm' | 'fastgpt' | 'n8n'

interface AgentConfig {
  id: string
  name: string
  icon: string
  description: string
  backend_type: BackendType
  api_key: string
  api_url: string
  model?: string               // direct_llm 模式必填
  is_default: boolean
  is_enabled: boolean
  extra_config?: Record<string, any>
}

type AgentInfo = Omit<AgentConfig, 'api_key' | 'api_url'>
```

### 3.4 localStorage 布局

| Key | 结构 | 说明 |
|-----|------|------|
| `open_chat_conversations` | `ConversationRecord[]` | 所有会话（含每 Agent 的参数和 backend convId） |
| `open_chat_messages` | `MessageRecord[]` | 所有消息（通过 conversation_id 关联） |
| `conversationIdInfo` | `{ [appId]: convId }` | 最后选中的会话 ID |

## 五、服务层

### 5.1 ConversationService

- `getAllConversations()` - 获取所有对话
- `createConversation(name, id?)` - 创建新对话（自动初始化 `agents: {}`）
- `updateConversation(id, updates)` - 更新对话（name 或 agents）
- `deleteConversation(id)` - 删除对话及其消息

**Agent Session 管理：**
- `getAgentSession(convId, agentId)` - 获取 Agent 在本会话中的状态
- `saveAgentSession(convId, agentId, updates)` - 合并写入 Agent 状态
- `getAgentParams(convId, agentId)` - 获取已存参数（同步调用方使用 sync helper）
- `saveAgentParams(convId, agentId, params)` - 写入参数
- `getBackendConversationId(convId, agentId)` - 获取后端会话 ID
- `saveBackendConversationId(convId, agentId, id)` - 写入后端会话 ID

**前端 sync helper（内联在 Main 组件中）：**
由于 localStorage 操作本身是同步的，前端使用同步 helper 函数直接读写，避免 Promise 包装：
```
getAgentParamsSync(convId, agentId)            → 直接读 localStorage
saveAgentParamsSync(convId, agentId, params)   → 直接写 localStorage
getBackendConvIdSync(convId, agentId)          → 同步读
saveBackendConvIdSync(convId, agentId, id)     → 同步写
```

### 5.2 MessageService

- `getMessages(conversationId)` - 获取对话的消息列表（按时间排序）
- `saveMessage(msg)` - 保存消息
- `saveUserMessage(...)` - 保存用户消息
- `saveAssistantMessage(...)` - 保存 AI 回复

### 8.3 发送消息 API 示例

```typescript
// POST /api/chat-messages
// Body: { query, conversation_id, agent_id, ... }

export async function POST(request: NextRequest) {
  const body = await request.json()
  const { query, conversation_id, agent_id, ...rest } = body

  const agent = agent_id ? getAgentById(agent_id) : getDefaultAgent()
  const adapter = createAdapter(agent)
  const { user } = getInfo(request)

  const response = await adapter.sendMessage({
    query, conversation_id, user,
    inputs: rest.inputs, files: rest.files,
  })

  return new Response(response.body, {
    headers: { 'Content-Type': 'text/event-stream' },
  })
}
```

## 十、参数同步机制

### 10.1 核心不变式

```
表单渲染的值 == agentInputsCacheRef[agentKey] == localStorage conv.agents[agentKey].params
三者永远相等，无需任何"过滤"
```

### 10.2 参数定义获取策略

**懒加载 + 定时刷新：** 使用到哪个 Agent 才从服务端 fetch 其参数定义，取后缓存到 `promptVariablesCacheRef`。每次切换到该 Agent 时重新 fetch 最新定义，确保后台修改参数后前端及时感知。

| 时机 | 行为 | 同步/异步 |
|------|------|-----------|
| **进入页面** | fetch 默认 Agent 参数定义 → 同步清洗已存参数 → 缓存定义 → 渲染 | 首次异步 |
| **切换到 Agent** | fetch 最新参数定义 → 同步清洗 → 缓存 → 恢复表单 | 异步（始终从服务端拉） |

### 10.3 参数同步时机

#### ① 切换智能体
```
useEffect([selectedAgentId]) {
  if (!inited) return

  // 1. 同步保存旧 Agent 参数到 localStorage
  saveAgentParamsSync(convId, prevKey, currInputs)
  agentInputsCacheRef[prevKey] = currInputs

  // 2. 立即清空表单
  setCurrInputs(null)
  setPromptConfig(null)

  // 3. 始终从服务端 fetch 最新参数定义，然后同步清洗 + 恢复
  fetchAndCachePromptVars(agentKey).then(promptVars => {
    const cleaned = syncAndCleanParams(convId, agentKey, promptVars)
    setCurrInputs(cleaned ? { ...cleaned } : null)
    setPromptConfig({ prompt_variables: promptVars })
  })
}
```
- `setCurrInputs(null)` + `setCurrInputs(cleaned)` 在同一个 render 周期 → 原子切换，无闪烁
- 不再区分 cache 命中/未命中，每次切换都 fetch 最新定义
- `agentKey` 永远是实际的智能体 ID（不传时用默认智能体的实际 ID），不使用 `'__default__'`

#### ② 切换会话
```
loadAgentSpecificParams → syncAndCleanParams → setCurrInputs
```

#### ③ 表单输入变更（sync effect）
```
currInputs 变化 →
  agentInputsCacheRef[agentKey] = { ...currInputs }
  if (realConvId) saveAgentParamsSync(realConvId, agentKey, currInputs)
```
写入的内容与表单渲染完全一致。

#### ④ 发送消息
```
// 1. 从 cache 或 localStorage 同步取当前 Agent 参数（不再从 currInputs 取）
const resolvedInputs = cache[agentKey] || getAgentParamsSync(convId, agentKey) || {}

// 2. 验证必填项（按 promptConfig.prompt_variables）

// 3. 构建 toServerInputs ← resolvedInputs（不滤波）
// 4. 写回（保持三者一致）
cache[agentKey] = toServerInputs
saveAgentParamsSync(convId, agentKey, toServerInputs)
```

#### ⑤ Dify conversation_id 映射
```
第一次发送 → conversation_id: null
onData 回调 → saveBackendConvIdSync(convId, agentKey, difyConvId)  // 同步写
后续发送   → diffyConvId = getBackendConvIdSync(convId, agentKey)   // 同步读
```

### 10.4 syncAndCleanParams（参数清洗）

```
function syncAndCleanParams(convId, agentId, promptVars):
  1. getAgentParamsSync(convId, agentId) → 读已存参数
  2. 对比 promptVars 的 key 集合
     - validKeys = Set(promptVars.map(v => v.key))
     - 删除已存参数中不属于 validKeys 的 key（Agent 定义已变更）
     - 保留匹配的 key
  3. 如果有删除 → saveAgentParamsSync 写回 localStorage
  4. 更新 agentInputsCacheRef[agentId]
  5. return cleanedParams
```

### 10.5 数据流（完整）

```
页面加载
  ├─ fetchAgents → 取智能体列表 → setDefaultAgentId
  ├─ fetchAppParams → 取默认 Agent 的 prompt_variables → 缓存
  ├─ fetchConversations → 取本地会话列表
  └─ syncAndCleanParams → 清洗默认 Agent 参数 → setCurrInputs → 表单渲染

切换智能体
  ├─ saveAgentParamsSync(旧 Agent) → 保存
  ├─ setCurrInputs(null) + setPromptConfig(null) → 表单清空
  ├─ fetchAndCachePromptVars(agentKey) → 从服务端 fetch 最新定义
  ├─ syncAndCleanParams → 清洗新 Agent 参数
  └─ setCurrInputs(cleaned) + setPromptConfig(vars) → 表单更新

发送消息
  ├─ promptConfig 守卫（null 则阻塞，prompt_variables=[] 则放行）
  ├─ getAgentParamsSync → 取当前 Agent 参数
  ├─ 验证必填项（跳过无参数智能体）
  ├─ 构建 toServerInputs → 发送
  ├─ 写回 localStorage + cache
  └─ onData 回调 → saveBackendConvIdSync（保存 Dify convId）
```

### 10.6 agentKey 规则

- **永远使用实际智能体 ID**，不使用 `'__default__'` 魔术字符串
- `agentKey = selectedAgentId || defaultAgentId`（defaultAgentId 来自配置文件）
- 切换默认智能体 → key 变化 → 旧参数不会泄漏到新默认智能体

## 十一、数据流（消息发送）

```
用户发送消息
  ↓
① 同步取当前智能体已存参数 → 验证必填项
  ↓
② 前端携带 agent_id + 参数 调用 /api/chat-messages
  ↓
③ 后端根据 agent_id 获取 AgentConfig
  ↓
④ 根据 backend_type 创建对应适配器
  ↓
⑤ 适配器调用对应的后端 API（首次发送 conversation_id: null）
  ↓
⑥ 返回 SSE 流
  ↓
⑦ onData 回调：同步保存 backend_conversation_id 到 localStorage
  ↓
⑧ onCompleted：保存消息到本地存储（携带 agent_id）+ 更新会话标题
  ↓
⑨ 更新界面显示
```
