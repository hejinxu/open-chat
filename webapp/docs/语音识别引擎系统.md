# 语音识别引擎系统

本项目支持两种语音识别引擎：浏览器内置识别和 Whisper 服务端识别。

## 架构概述

### 浏览器识别引擎 (`BrowserRecognition`)

- 使用浏览器 Web Speech API
- 通过 `window.SpeechRecognition` / `window.webkitSpeechRecognition` 运行
- 需要 Chrome 浏览器联网（音频发送到 Google 服务器）
- 支持连续识别和中间结果，实时性强

### Whisper 服务端识别引擎 (`WhisperRecognition`)

- 浏览器端录音（`AudioContext` + `ScriptProcessor`），实时发送 16kHz Float32 采样数据
- 通过 Socket.IO 与独立的 WS Server 通信（namespace: `/speech`）
- 服务端累积音频缓冲区，增量处理（仅转写新增音频）
- 持续识别，支持长对话
- 无需浏览器联网，完全离线可用

```
浏览器                              WS Server (ws://localhost:8787/speech)
┌──────────────┐    Socket.IO     ┌──────────────────────────┐
│ AudioContext   │ ─── Float32 ──→ │ 累积音频缓冲区            │
│ ScriptProcessor │ ←── JSON ──── │ ↓ buffer ≥ 0.5s 时转写   │
│ (16kHz采样)     │    识别结果    │ Whisper Pipeline         │
└──────────────┘                  │ 结果去重 → 只发新结果     │
                                  └──────────────────────────┘
                                           ↓
                                    ws-server/models/
                                    onnx-community/
                                    whisper-small/
```

## 引擎切换

用户可以在语音设置面板中选择识别引擎：

- **浏览器识别**（默认）：速度快，延迟低，但需要联网
- **Whisper 离线**：服务端推理，无需联网，支持长对话持续识别

引擎选择保存在 `localStorage` 的 `voice-engine` 键中。

## 启动方式

### 同时启动两个服务

```bash
# 终端1：启动 Next.js webapp
pnpm dev

# 终端2：启动 WS Server
pnpm ws-server
```

### WS Server 环境变量

| 变量 | 默认值 | 说明 |
|---|---|---|
| `WS_PORT` | `8787` | WebSocket 端口 |
| `SPEECH_MODEL` | `whisper-tiny` | 默认模型 |
| `SPEECH_MODEL_PATH` | `./models` | 模型文件路径 |
| `SPEECH_PROCESS_INTERVAL` | `1500` | 推理间隔（毫秒） |
| `SPEECH_MIN_AUDIO_LENGTH` | `8000` | 最小音频长度（16kHz下0.5秒=8000采样） |
| `SPEECH_OFFLINE` | - | 设置后禁止远程下载模型 |
| `SPEECH_MIRROR` | - | 设置模型下载镜像地址 |

## 配置项

**文件**: `config/voice-input.ts`

```typescript
export type VoiceRecognitionEngine = 'browser' | 'whisper'

export const VOICE_INPUT_CONFIG = {
  NO_INPUT_TIMEOUT_MS_BROWSER: 5000,
  NO_INPUT_TIMEOUT_MS_WHISPER: 8000,
  AUTO_SEND_ON_STOP: true,
  AUTO_READ_ALOUD: true,
  DEFAULT_ENGINE: 'browser' as VoiceRecognitionEngine,
  WHISPER_MODEL: 'onnx-community/whisper-small',
  SEND_DELAY_MS: 5000,
}
```

### localStorage 键

| 键 | 说明 | 类型 |
|---|---|---|
| `voice-engine` | 识别引擎选择 | `'browser'` / `'whisper'` |
| `voice-auto-stop-on-no-input` | 无录音输入自动结束 | `'true'` / `'false'` |
| `voice-auto-send-on-stop` | 录音结束自动发送 | `'true'` / `'false'` |
| `voice-auto-read` | 自动朗读回复 | `'true'` / `'false'` |
| `voice-no-input-ms-browser` | 浏览器识别无录音时长（毫秒） | 数字字符串 |
| `voice-no-input-ms-whisper` | Whisper 无录音时长（毫秒） | 数字字符串 |

## Socket.IO 协议

### Namespace

- `/speech` - 语音识别服务
- `/push` - 后端推送服务（预留）

### Speech 事件

**客户端 → 服务端：**
```json
{ "type": "config", "model": "whisper-tiny" }  // 设置模型
{ "type": "audio", "data": [...] }              // Float32Array 采样数据
{ "type": "stop" }                              // 停止识别
```

**服务端 → 客户端：**
```json
{ "type": "ready" }                                       // 连接就绪
{ "type": "config_ok", "model": "whisper-tiny" }          // 模型设置成功
{ "type": "result", "text": "识别结果", "is_final": true } // 识别结果
{ "type": "stopped" }                                      // 已停止
{ "type": "error", "message": "错误信息" }                    // 错误
```

## 服务端处理逻辑

### 转写与结果去重

Whisper 服务端采用"完整 buffer + 结果去重"策略：

1. **累积音频**：音频数据不断追加到 `audioBuffer`，保留完整上下文供 Whisper 识别
2. **触发转写**：当 `buffer.length >= MIN_AUDIO_LENGTH` (8000 samples = 0.5s) 且未在处理中时触发
3. **转写整个 buffer**：Whisper 需要完整上下文才能正确识别
4. **结果去重**：只在转写结果与上次不同时才发送给客户端，避免重复结果重置客户端 timer
5. **静音跳过**：RMS < 0.03 的 buffer 跳过转写
6. **buffer 清理**：仅在收到 `stop` 消息时清空 buffer

```
音频数据到达 → 追加到 buffer
    ↓
buffer.length ≥ MIN_AUDIO_LENGTH？
    ↓ 是
转写整个 buffer → 结果与上次不同？
    ↓ 是
发送结果到客户端 → 客户端重置 speechTimerRef
    ↓
用户停止说话 → 无新结果 → timer 正常触发 → 自动停止
```

### 自动停止流程

1. 用户停止说话 → 无新音频到达
2. buffer 不再增长 → 但 `processTimeout` 仍会周期性触发转写
3. 转写结果与上次相同 → 去重过滤 → 不发送结果
4. 客户端 speechTimerRef 不被重置 → 正常倒计时
5. 倒计时结束 → 录音自动停止

### 静音检测

- **RMS 阈值**：`SILENCE_THRESHOLD = 0.03`
- **`processBuffer`**：转写前检查 RMS，低于阈值则跳过
- **`processTimeout`**：定时器触发时也检查 RMS，低于阈值则不转写
