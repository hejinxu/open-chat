## 主题系统架构概述

项目采用 **CSS Custom Properties** 方案实现多主题：

```
app/styles/
├── globals.css              ← 基础样式 + 导入主题文件
└── themes/
    ├── light.css            ← :root 变量定义（默认值）
    ├── dark.css             ← .dark 变量覆盖
    └── tech-blue.css        ← .tech-blue 变量覆盖

tailwind.config.js           ← 语义化颜色映射（引用 CSS 变量）
config/theme.ts              ← 主题类型定义
hooks/use-theme.ts           ← 主题状态管理（切换 class）
```

### 工作原理

1. 每个主题文件在对应 class 选择器下定义一套 CSS 变量（如 `--surface`、`--text`、`--border`）
2. Tailwind 配置将语义化类名映射到这些变量（如 `bg-surface` → `var(--surface)`）
3. 组件使用语义化类名（如 `bg-surface text-content border-border`）
4. `useTheme` Hook 在 `<html>` 上切换 class，CSS 变量自动生效

## 步骤 1：创建主题 CSS 变量文件

在 `app/styles/themes/` 目录下创建新主题文件（如 `ocean.css`）：

  --text-quaternary: #6090B8;
  --text-accent: #00B4FF;

## 步骤 2：更新主题配置

在 `config/theme.ts` 中添加新主题：

```typescript
// config/theme.ts

export type ThemeMode = 'light' | 'dark' | 'system' | 'tech-blue' | 'ocean'

export const THEME_MODES: Record<string, ThemeMode> = {
  LIGHT: 'light',
  DARK: 'dark',
  SYSTEM: 'system',
  TECH_BLUE: 'tech-blue',
  OCEAN: 'ocean',  // 新增
}
```

## 步骤 3：更新 Tailwind 配置

确保 `tailwind.config.js` 中有对应的语义化颜色映射（通常无需修改，除非需要新增颜色类别）：

```javascript
// tailwind.config.js
theme: {
  extend: {
    colors: {
      surface: {
        DEFAULT: 'var(--surface)',
        secondary: 'var(--surface-secondary)',
        tertiary: 'var(--surface-tertiary)',
        hover: 'var(--surface-hover)',
        active: 'var(--surface-active)',
        elevated: 'var(--surface-elevated)',
      },
      content: {
        DEFAULT: 'var(--text)',
        secondary: 'var(--text-secondary)',
        tertiary: 'var(--text-tertiary)',
        quaternary: 'var(--text-quaternary)',
        accent: 'var(--text-accent)',
        'on-accent': 'var(--text-on-accent)',
      },
      border: {
        DEFAULT: 'var(--border)',
        subtle: 'var(--border-subtle)',
        strong: 'var(--border-strong)',
        accent: 'var(--accent)',
        'accent-hover': 'var(--accent-hover)',
      },
      accent: {
        DEFAULT: 'var(--accent)',
        hover: 'var(--accent-hover)',
        bg: 'var(--accent-bg)',
        'bg-hover': 'var(--accent-bg-hover)',
      },
      'user-bubble': {
        DEFAULT: 'var(--user-bubble)',
        text: 'var(--user-bubble-text)',
      },
      fill: {
        DEFAULT: 'var(--fill)',
        light: 'var(--fill-light)',
        'on-accent': 'var(--fill-on-accent)',
      },
    },
  },
},
```

## 步骤 5：添加主题切换选项

在 `app/components/theme-toggle-button/index.tsx` 中添加新主题选项：

```tsx
import { THEME_MODES } from '@/config/theme'
import { MoonIcon, SunIcon, ComputerDesktopIcon, SparklesIcon, WavesIcon } from '@heroicons/react/24/outline'

// 添加图标
const getThemeIcon = () => {
  if (theme === THEME_MODES.LIGHT) return <SunIcon className="h-4 w-4" />
  if (theme === THEME_MODES.DARK) return <MoonIcon className="h-4 w-4" />
  if (theme === THEME_MODES.TECH_BLUE) return <SparklesIcon className="h-4 w-4" />
  if (theme === THEME_MODES.OCEAN) return <WavesIcon className="h-4 w-4" />  // 新增
  return <ComputerDesktopIcon className="h-4 w-4" />
}

// 在下拉菜单中添加选项
<button onClick={() => handleDropdownItemClick(THEME_MODES.OCEAN)}>
  <WavesIcon className="h-4 w-4" />
  海洋蓝
</button>
```

## 步骤 6：更新 use-theme Hook

在 `hooks/use-theme.ts` 中更新 `toggleTheme` 函数：

```typescript
const toggleTheme = () => {
  const nextTheme = theme === THEME_MODES.LIGHT
    ? THEME_MODES.DARK
    : theme === THEME_MODES.DARK
      ? THEME_MODES.TECH_BLUE
      : theme === THEME_MODES.TECH_BLUE
        ? THEME_MODES.OCEAN  // 新增
        : theme === THEME_MODES.OCEAN
          ? THEME_MODES.SYSTEM
          : THEME_MODES.LIGHT
  setTheme(nextTheme)
}
```

同时更新 `resolvedTheme` 类型：

```typescript
export interface UseThemeReturn {
  theme: ThemeMode
  resolvedTheme: 'light' | 'dark' | 'tech-blue' | 'ocean'  // 新增
  setTheme: (theme: ThemeMode) => void
  toggleTheme: () => void
}
```

## 组件开发规范

### 使用语义化类名

组件中应使用语义化 Tailwind 类名，而非硬编码颜色或 `dark:` 前缀：

| 用途 | ✅ 正确写法 | ❌ 错误写法 |
|------|------------|------------|
| 背景色 | `bg-surface` | `bg-white dark:bg-gray-800` |
| 文字色 | `text-content` | `text-gray-900 dark:text-white` |
| 边框色 | `border-border` | `border-gray-200 dark:border-gray-700` |
| 悬停背景 | `hover:bg-surface-hover` | `hover:bg-gray-100 dark:hover:bg-gray-700` |
| 强调色 | `text-content-accent` | `text-primary-600 dark:text-primary-400` |
| 选中背景 | `bg-accent-bg` | `bg-primary-50 dark:bg-primary-900/30` |

### 弹出层使用 elevated 背景

下拉菜单、tooltip、弹窗等浮层应使用 `bg-surface-elevated`（完全不透明），避免半透明背景导致透视问题：

```tsx
// ✅ 正确
<div className="bg-surface-elevated rounded-lg shadow-lg border border-border">

// ❌ 错误（半透明背景会透视）
<div className="bg-surface rounded-lg shadow-lg">
```

### Focus 样式使用 CSS 变量

输入框的 focus 环颜色通过 `--ring` 变量控制：

```tsx
// ✅ 正确
<input
  className="border border-border focus:outline-none focus:ring-2"
  style={{ '--tw-ring-color': 'var(--ring)' } as React.CSSProperties}
/>

// ❌ 错误（硬编码颜色）
<input className="border border-gray-200 focus:ring-blue-500" />
```

### 用户消息气泡

用户消息气泡使用独立的 `--user-bubble` 变量：

```tsx
// ✅ 正确
<div className="bg-user-bubble text-user-bubble-text rounded-tl-2xl rounded-b-2xl">

// ❌ 错误（硬编码颜色）
<div className="bg-blue-500 text-white rounded-tl-2xl rounded-b-2xl">
```

### 常见问题

**Q：颜色没有生效？**
1. 检查主题 CSS 文件是否在 `globals.css` 中正确导入
2. 检查变量名是否与 Tailwind 配置中的映射一致
3. 清除浏览器缓存和 `localStorage.removeItem('theme-mode')`
4. 重启开发服务器

**Q：弹出层背景半透明？**
弹出层（下拉菜单、tooltip、弹窗）必须使用 `bg-surface-elevated`，不能使用 `bg-surface`。

**Q：Focus 环颜色不对？**
检查是否使用了 `style={{ '--tw-ring-color': 'var(--ring)' }}`，且主题文件中定义了 `--ring` 变量。

**Q：需要修改已有组件？**
只需确保组件使用语义化类名。如果组件使用了硬编码颜色（如 `bg-white`、`text-gray-900`），需要替换为对应的语义化类名。
